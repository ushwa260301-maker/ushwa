## 한 줄 정의

대한민국 조경업체의 거래명세서를 **공용 데이터베이스**로 축적하고, 그
데이터로 **OCR 인식 엔진을 지속적으로 자가 개선**하는 조경 AI 플랫폼.

## 핵심 철학 (확정)

1. **코드보다 데이터가 중요하다.**
   - 기능을 늘리는 것이 아니라, 실제 거래명세서·OCR 정답·수정 이력이
     계속 쌓이는 구조를 만드는 것이 목표다.

2. **OCR 은 거래명세서를 "읽는" 기능이 아니라, "AI 학습 데이터를
   생성하는" 엔진이다.**
   - 사용자가 OCR 결과를 수정할 때마다 `parsed_fields` ↔
     `user_edited_fields` 쌍이 학습 데이터로 축적된다.
   - 이 축적 데이터가 다음 OCR 엔진 개선의 근거가 된다.

3. **데이터는 공용이다. 개인 소유권이 없다.**
   - Google 로그인은 "누가 올렸는지" 기록(attribution)일 뿐, 데이터
     접근을 사용자별로 격리하지 않는다.
   - 로그인한 모든 사용자는 동일한 거래명세서·OCR 결과·수종 데이터를
     본다.

4. **Cloud Database 가 Single Source of Truth 다.**
   - LocalStorage 는 캐시, IndexedDB 는 첨부 파일 캐시.
   - Cloud 가 원본이며, 캐시는 항상 Cloud 로 덮인다.

5. **Fixture 는 삭제하지 않는다.**
   - 실제 OCR 실패 사례는 회귀 테스트 자산으로 영구 누적한다.
   - 기존 fixture 를 수정하는 대신 새 fixture 를 추가한다.

6. **기존 기능을 절대 깨뜨리지 않는다.**
   - 모든 변경은 회귀 테스트(현재 OCR fixture 229/240)를 통과해야 한다.

## 품질 목표 (확정)

- **OCR 인식률 99%+** — 실제 한국 조경업체 거래명세서 기준.
- 사람이 수정해야 하는 항목을 지속적으로 줄이는 것이 성공 지표다.
- Parser 수정은 전체 수정의 20% 이하를 목표로 하고, 같은 유형 실패가
  3개 이상 fixture 에서 반복되면 OCR Engine 개선으로 전환한다.

## 사용자 (확정)

- 조경업체 실무자. Google 로그인으로 접속.
- 거래명세서(JPG/PNG/PDF)를 업로드 → OCR → 검토·수정 → 저장.
