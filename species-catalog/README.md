# 수종 도감 (Species Catalog)

식물 수종별 **개화시기 · 단가표 · 수급처**를 데이터로 축적하고, 필터·검색·정렬로 조회하는 정적 웹앱입니다.
브라우저에서 직접 **추가·수정·삭제** 가능하며, 저장은 localStorage에, 백업/공유는 **JSON 파일 내보내기/가져오기**로 처리합니다.

어서화(꽃 중개 플랫폼)와 완전히 분리된 독립 모듈로, 서버·빌드 도구 없이 정적 서버만 있으면 동작합니다.

## 폴더 구조

```
species-catalog/
├── index.html          # UI: masthead + 필터 rail + 카드 그리드 + 편집 모달 + 명세서 임포터
├── css/style.css       # 스타일 (라이트/다크 자동)
├── js/app.js           # 상태·필터·CRUD·저장·JSON 임포트/익스포트
├── js/importer.js      # 거래명세서(이미지/PDF) → 대조표 반영 (Tesseract.js + PDF.js)
├── data/species.json   # 최초 시드 (12개 예시)
└── README.md
```

## 실행

`fetch()`로 시드 JSON을 읽으므로 로컬 서버로 열어야 합니다.

```bash
cd species-catalog
python3 -m http.server 8080
# 브라우저에서 http://localhost:8080
```

또는 VS Code Live Server, `npx serve` 등 정적 서버 아무 것이나 무방합니다.

## 데이터 관리

### 최초 로드
1. `localStorage["species-catalog:v1"]`가 있으면 → 그 값을 씀
2. 없으면 → `data/species.json`을 시드로 저장

### 추가·수정·삭제
- 우상단 **+ 수종 추가** 버튼으로 모달을 열어 신규 등록
- 카드에 마우스를 올리면 우상단에 **✎(수정)**, **✕(삭제)** 아이콘이 나타남
- 저장 즉시 localStorage에 반영

### 백업 / 공유
- **내보내기**: 현재 데이터를 `species-catalog-YYYY-MM-DD.json` 파일로 다운로드
- **가져오기**: 저장한 JSON 파일을 업로드 → 현재 데이터를 대체
- **시드 복원**: 모든 사용자 데이터를 지우고 `data/species.json`으로 리셋

### 단일 수종 등록 시 명세서 첨부 (권장)
1. 툴바의 **+ 수종 추가** 클릭 → 모달 최상단 **명세서 첨부** 섹션
2. 이미지·PDF 파일 첨부
3. 진행률 표시와 함께 OCR·PDF 텍스트 추출 → 자동 파싱
4. 인식된 값이 폼에 자동 채워짐:
   - **상호·소재지·핸드폰** → 수급처 첫 행
   - **첫 품목명** → 수종명
   - **모든 (규격·단위·단가)** → 단가표 행들
5. 사용자가 각 필드 검토·수정 (개화월·색상·학명 등 추가 정보 보완)
6. **저장** 클릭 → 대조표에 등록

이 방식이 손글씨/저품질 문서에도 유용합니다 — OCR이 실패해도 사용자가 직접 이어서 입력하면 됩니다.

### 거래명세서 대량 임포트 (여러 품목 한번에)
1. 툴바의 **명세서 가져오기** 버튼 클릭
2. 이미지(JPG/PNG) 또는 PDF 파일 선택
3. 자동으로 텍스트 추출:
   - PDF에 내장 텍스트가 있으면 PDF.js가 바로 파싱
   - 스캔 PDF·이미지면 Tesseract.js가 한국어+영어 OCR 실행 (최초 실행 시 언어 데이터 ~10MB 다운로드)
4. 추출된 텍스트를 직접 수정 가능 → **자동 파싱** 클릭
5. 인식된 항목 표에서:
   - 체크박스로 반영할 행 선택
   - 각 칼럼(품목·규격·단위·단가) 인라인 수정
   - **대상 수종** 드롭다운으로 기존 수종에 추가하거나 새로 생성 선택
6. 수급처 정보 확인/수정 후 **선택 항목 반영**

#### 반영 필수 항목 (검증 규칙)
반영 시 다음 6개 필드가 모두 채워져 있어야 합니다:

| 항목 | 대상 필드 | 자동 감지 방법 |
|---|---|---|
| **상호** | 수급처 이름 | `상호:`/`업체명:`/`공급자:` 라벨 우선 → 회사형 접미어(농원·수목원·원예·농장·조합·㈜ 등) → 첫 유의미한 한글 라인 |
| **사업장 소재지** | 수급처 지역 | `주소:`/`소재지:`/`사업장 소재지:` 라벨 우선 → 시/도 + 시/군/구 + 동/면/리/로/길 조합 |
| **핸드폰 번호** | 수급처 연락처 | `핸드폰:`/`휴대전화:`/`HP:` 라벨 우선 → 010·011·016~019 접두 → 일반 전화번호 |
| **품목** | 수종명 | 라인 앞부분의 한글 명사 (2~25자) |
| **규격** | 단가표 spec | `R\d+`, `H\d+.?\d*`, `\d+분`, `\d+포트`, `\d+주` 등 조경업계 관례 |
| **단가** | 단가표 price | 콤마 포함 숫자 또는 `원` 접미어, 4자리 이상 |

- **수급처 3개(상호·소재지·핸드폰)** 중 하나라도 비어 있으면 반영이 차단됩니다.
- **품목·규격·단가** 중 하나라도 비어 있으면 그 행은 자동 제외되고(붉게 표시) 확인 후 진행됩니다.
- 기존 수종에 반영할 때 같은 상호+연락처 수급처가 이미 있으면 중복 추가되지 않습니다.

**정확도 안내**: 클라이언트 OCR은 문서 화질과 형식에 따라 오인식이 잦습니다. 반영 전 반드시 검토하세요. 특히 손글씨·저해상도·복잡한 표는 정확도가 크게 떨어집니다.

### 마스터 목록 (카테고리·색상)
- 모달에서 카테고리는 드롭다운으로 선택하거나 **새 카테고리** 입력창에 타이핑
- 색상은 기존 칩 클릭 또는 하단 입력창에 새 값 추가
- 새로 넣은 값은 자동으로 마스터에 편입되어 이후 필터·모달에서 재사용

## 필터

| 필터 | 동작 |
|---|---|
| 검색 | 수종명/학명 부분일치 (대소문자 무시) |
| 개화월 | 다중 선택. 하나라도 겹치면 매칭 |
| 카테고리 | 다중 선택 (OR) |
| 개화색 | 다중 선택. 하나라도 겹치면 매칭 |
| 수급처 | 단일 선택. 데이터에서 자동 수집 |
| 가격 범위 | 단가표 항목 중 하나라도 범위 내면 매칭 |

## 데이터 스키마

```jsonc
{
  "categories": ["교목", "관목", ...],
  "colors": ["백색", "황색", ...],
  "species": [
    {
      "id": "sp-001",                        // 자동 부여 (편집 시 유지)
      "name": "왕벚나무",                    // 필수
      "scientificName": "Prunus × yedoensis",
      "category": "교목",                    // categories 중 하나
      "bloomMonths": [3, 4],                 // 1~12
      "colors": ["백색", "분홍"],
      "prices": [
        { "spec": "R6", "unit": "주", "price": 45000 }
      ],
      "suppliers": [
        { "name": "천리포수목원", "region": "충남 태안", "contact": "041-672-9982" }
      ],
      "notes": "가로수·공원용으로 다용."
    }
  ]
}
```

## 참고

`data/species.json`의 시드 데이터(수종·가격·수급처)는 **구조 예시용 목업**입니다. 실제 시세와 실 취급 여부는 검증되지 않았으므로 실무 발주에는 사용하지 마세요. 정확한 데이터는 국립수목원, 조달청 나라장터, 지역 산림조합 등의 자료로 대체하시기 바랍니다.
